# Database Schema

The database schema is defined in `stacker-api/prisma/schema.prisma` and uses MariaDB with the Prisma MariaDB driver adapter.

## Models

### Users & Permissions

```
users
  id                           VARCHAR(100) PK, UUID
  email                        VARCHAR(255) UNIQUE
  password_hash                VARCHAR(255) nullable
  first_name                   VARCHAR(100) nullable
  last_name                    VARCHAR(100) nullable
  email_verified_at            DATETIME nullable
  email_verification_token     VARCHAR(255) UNIQUE nullable
  email_verification_expires_at DATETIME nullable
  password_reset_token         VARCHAR(255) UNIQUE nullable
  password_reset_expires_at    DATETIME nullable
  created_at                   DATETIME default now()
  updated_at                   DATETIME auto-updated

groups
  id                           CHAR(36) PK, UUID
  name                         VARCHAR(200)
  slug                         VARCHAR(200) UNIQUE
  description                  VARCHAR(500) nullable
  is_system                    BOOLEAN default false
  sort_order                   INT default 0
  created_at / updated_at      DATETIME

group_permissions
  group_id + permission        Composite PK
  permission                   VARCHAR(100)

user_groups
  user_id + group_id           Composite PK
```

### Domain (Bookshelf)

```
authors
  id                           CHAR(36) PK, UUID
  name                         VARCHAR(200)
  bio                          TEXT nullable
  created_at / updated_at      DATETIME

books
  id                           CHAR(36) PK, UUID
  title                        VARCHAR(300)
  author_id                    CHAR(36) FK -> authors
  genre                        VARCHAR(100) nullable
  description                  TEXT nullable
  cover_url                    VARCHAR(500) nullable
  created_at / updated_at      DATETIME

reviews
  id                           CHAR(36) PK, UUID
  book_id                      CHAR(36) FK -> books
  user_id                      VARCHAR(100) FK -> users
  rating                       SMALLINT (1-5)
  text                         TEXT nullable
  created_at / updated_at      DATETIME
  UNIQUE(book_id, user_id)     One review per user per book
```

## Key Design Decisions

### UUIDs

All primary keys are UUIDs generated by Prisma's `@default(uuid())`. This avoids auto-increment leakage and makes IDs portable across environments.

### Cascade Deletes

All foreign keys use `onDelete: Cascade`. Deleting an author deletes their books; deleting a book deletes its reviews; deleting a group deletes its permission entries and memberships.

### Prisma MariaDB Adapter

Stacker uses `@prisma/adapter-mariadb` instead of Prisma's built-in MySQL support. The adapter connects via the `DATABASE_URL` connection string. The Prisma client is a singleton in `src/context/prisma.ts`.

### Barrel Export

After `prisma generate`, a barrel file at `src/generated/prisma/index.ts` re-exports the client, enums, models, and common input types. This is generated by `scripts/write-prisma-barrel.js` and allows cleaner imports throughout the codebase.
