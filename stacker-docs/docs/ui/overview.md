# UI Overview

The Stacker UI is a React single-page application built with Vite and TypeScript.

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Framework | React 19 |
| Build Tool | Vite 8 |
| Language | TypeScript 6 |
| Data | Apollo Client 4 (GraphQL) |
| Components | shadcn/ui + Radix UI + Tailwind CSS |
| Forms | React Hook Form |
| Routing | React Router v7 |
| Testing | Playwright (E2E) |

## Architecture

### Provider Composition

The app wraps all routes in a provider chain:

```
ApolloProviderWrapper -> Suspense -> Routes
```

`ApolloProviderWrapper` configures the Apollo Client with three links chained together:

1. **omitTypenameLink** -- strips `__typename` from mutation variables (prevents cache conflicts)
2. **authLink** -- injects the JWT token in the `Authorization` header
3. **httpLink** -- sends requests to the GraphQL API

### Page Loading

All pages are lazy-loaded with `React.lazy()` and wrapped in `Suspense` with a spinner fallback. Only `SignInPage` is eagerly loaded (it's the first page users see).

### Auth State

Auth state (JWT token, user object) is stored in `localStorage` and managed by `AuthProvider` via React Context. The `useAuth()` hook provides:

- `isSignedIn` / `isLoaded` -- auth status
- `token` / `user` / `userId` -- current session
- `signIn(token, user)` / `signOut()` -- auth actions

### Permission Checks

The `usePermissions()` hook fetches the current user's permissions from the `myPermissions` GraphQL query. The `RequirePermission` component uses this to gate route access:

```tsx
<RequirePermission permission="books.manage">
  <BookFormPage />
</RequirePermission>
```

### Runtime Environment

For Docker deployments, environment variables are injected at container startup via `window.__ENV__` (generated by `docker-entrypoint.sh`). The `env()` helper checks `window.__ENV__` first, then falls back to Vite's `import.meta.env`.

### XSS Protection

The `SafeHtml` component renders user-generated HTML after sanitizing it with DOMPurify. Use this instead of `dangerouslySetInnerHTML` for any content from the database.

### Error Handling

The `extractErrorMessage()` utility normalizes Apollo GraphQL errors, standard Error objects, and unknown types into user-friendly strings. Used throughout the app for toast notifications on mutation errors.
